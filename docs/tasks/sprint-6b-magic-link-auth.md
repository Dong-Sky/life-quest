# Sprint 6B：用户名与密码登录

## 目标

用户可用用户名和密码注册、登录 Questline，并在不同设备持续使用自己的账号。第一版不发送邮件验证或登录链接。

## 包含

- 用户名与密码注册、登录
- 用户名到 Supabase 内部登录标识的稳定映射
- 首次注册自动创建 Supabase 用户与 profile
- 顶层会话确认与退出登录
- 用户名与密码输入的前端校验
- 用户名工具的单元测试
- Supabase 控制台设置与人工验收说明

## 不包含

- 忘记密码和密码找回
- 云端任务、主线、奖励的读取与写入
- 旧浏览器数据导入
- 共同副本和账号间共享

## 需要的控制台设置

合并后在 Supabase 的 **Authentication → Sign In / Providers → Email** 设置中关闭 **Confirm email**。否则 Supabase 会要求真实邮箱验证，与本轮“不发邮件”的体验冲突。
